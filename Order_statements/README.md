# Order_statements — автоматическая отправка заявлений в Росреестр

Скрипт открывает сайт Росреестра в Chrome, входит под вашей учётной записью и по очереди обрабатывает CSV-файлы из папки `uploads_CSV`: заполняет формы, подписывает заявления и отправляет их.

Ручной запуск — через `start.bat`.  
Автозапуск по расписанию (в том числе с пробуждением ПК из сна) — через `schedule.ini` и `setup_schedule.bat`.

## Первый запуск

### 1. Python и зависимости

В корне репозитория (`c:\Selenium`) должен быть виртуальное окружение `venv`. Если его нет:

```bat
cd c:\Selenium
python -m venv venv
venv\Scripts\pip install -r Order_statements\requirements.txt
```

`start.bat` и остальные bat-файлы автоматически используют `..\venv\Scripts\python.exe`.

### 2. Файл настроек `.env`

Переменные окружения читаются из:

- `Order_statements\.env` (приоритет)
- `files\.env` (общий для проекта)

Минимально нужны:

| Переменная | Назначение |
|------------|------------|
| `EMAIL` | Логин на сайте Росреестра |
| `CHROME_PATH` | Путь к `chrome.exe` |
| `CHROME_PROFILE_PATH` | Папка профиля Chrome (с сохранённой сессией) |
| `DRIVER_PATH` | Путь к `chromedriver.exe` |
| `DOCUMENT_NUMBER`, `DOCUMENT_DATE`, `ISSUING_AUTHORITY` | Реквизиты документа заявителя |
| `MIN_ADDRESS` | Адрес (минимальный вариант для автозаполнения) |
| `PDF_FILE_NAME`, `SIGNATURE_FILE_NAME` | Имена файлов в папке `uploads` |
| `QWART_EMAIL` | E-mail для уведомлений (если используется) |
| `CORRECTION` | Флаг/текст корректировки (если нужен) |

Пример (пути подставьте свои):

```env
EMAIL=user@example.com
CHROME_PATH=C:\Program Files\Google\Chrome\Application\chrome.exe
CHROME_PROFILE_PATH=C:\Selenium\ChromeProfile
DRIVER_PATH=C:\Selenium\chromedriver.exe
DOCUMENT_NUMBER=1234 567890
DOCUMENT_DATE=01.01.2020
ISSUING_AUTHORITY=УФМС ...
MIN_ADDRESS=г Москва, ул Примерная, д 1
PDF_FILE_NAME=Хадиков.pdf
SIGNATURE_FILE_NAME=Хадиков.pdf.sig
```

### 3. Проверка

Положите один тестовый CSV в `uploads_CSV`, запустите `start.bat` и убедитесь, что скрипт проходит вход и начинает обработку.

## Подготовка данных

| Папка | Что класть |
|-------|------------|
| `future_uploads\` | Очередь CSV для автоматической подготовки (основной способ при автозапуске) |
| `uploads_CSV\` | CSV для обработки в текущем запуске (можно положить вручную) |
| `uploads\` | PDF и файл подписи (`.sig`), имена — как в `.env` |

### Автоматическая очередь (`future_uploads`)

При каждом запуске (`start.bat` или планировщик) скрипт:

1. Проверяет, пуста ли папка `uploads_CSV`.
2. Если пуста — копирует из `future_uploads` следующие по порядку CSV, которые ещё не были успешно отправлены.
3. Уже отправленные файлы определяются по `results.log` (строки `УСПЕХ:`) — отдельный список не нужен.
4. Обрабатывает все CSV в `uploads_CSV`; после успешной отправки файл удаляется из `uploads_CSV`, оригинал в `future_uploads` остаётся.

**Ручной режим:** положите CSV сразу в `uploads_CSV` — подготовка из `future_uploads` не выполняется, пока там есть файлы.

**Лимиты** (в `schedule.ini`, действуют и при ручном запуске):

| Параметр | Значение | Описание |
|----------|----------|----------|
| `files_per_run` | `0` | Сколько файлов копировать за один запуск (`0` = все доступные) |
| `daily_limit` | `0` | Максимум успешных отправок в сутки (`0` = без лимита) |

Пример: 5 файлов в день по будням:

```ini
files_per_run=5
daily_limit=5
days=weekdays
```

Скрипт перебирает все файлы с расширением `.csv` в `uploads_CSV`. Успешно отправленные файлы удаляются с диска; ошибки фиксируются в логах.

Перед автозапуском достаточно положить CSV в `future_uploads` — в `uploads_CSV` класть не обязательно.

## Ручной запуск

Дважды щёлкните **`start.bat`** или выполните в командной строке:

```bat
cd c:\Selenium\Order_statements
start.bat
```

Что происходит:

1. Запускается `python -m script.main`
2. Открывается Chrome с вашим профилем
3. Выполняется вход на сайт
4. Обрабатываются все CSV из `uploads_CSV`
5. Окно остаётся открытым до нажатия клавиши (`pause`) — удобно смотреть результат

При ошибке смотрите `logs\errors.log` и `logs\diagnostics\`.

## Автозапуск по расписанию

Автозапуск настраивается через файл **`schedule.ini`** и применяется bat-файлом **`setup_schedule.bat`**.  
В Планировщике заданий Windows создаётся задача **`OrderStatements_Rosreestr`** с опцией **пробуждения компьютера из сна**.

### Файлы

| Файл | Назначение |
|------|------------|
| `schedule.ini` | Время, дни недели, вкл/выкл — редактируется вручную |
| `setup_schedule.bat` | Читает `schedule.ini` и создаёт/обновляет задание в планировщике |
| `disable_schedule.bat` | Быстро отключает автозапуск (`enabled=false` + удаление задания) |
| `run_scheduled.bat` | Запуск без паузы; вызывается планировщиком, не вручную |

### Параметры `schedule.ini`

```ini
enabled=false
time=08:00
days=weekdays
```

| Параметр | Значения | Описание |
|----------|----------|----------|
| `enabled` | `true` / `false` | `true` — расписание активно; `false` — задание удаляется из планировщика |
| `time` | `ЧЧ:ММ` | Время запуска в 24-часовом формате, например `07:30` |
| `days` | см. ниже | В какие дни запускать |
| `files_per_run` | число, `0` = все | Сколько CSV копировать из `future_uploads` за запуск |
| `daily_limit` | число, `0` = без лимита | Максимум успешных отправок в сутки |

**Варианты `days`:**

| Значение | Расписание |
|----------|------------|
| `daily` | Каждый день |
| `weekdays` | Понедельник — пятница |
| `пн,ср,пт` | Только указанные дни |
| `mon,wed,fri` | То же на английском |

Допустимые сокращения дней: `пн` `вт` `ср` `чт` `пт` `сб` `вс` (или `mon` … `sun`).

Строки, начинающиеся с `;`, — комментарии.

---

### Включить автозапуск

1. Откройте `schedule.ini` в блокноте.
2. Установите нужные значения, например:

   ```ini
   enabled=true
   time=08:30
   days=weekdays
   ```

3. Сохраните файл.
4. Запустите **`setup_schedule.bat`** **от имени администратора** (правый щелчок → «Запуск от имени администратора»).

   Администратор нужен, чтобы включить таймеры пробуждения в схеме питания Windows.

5. В окне должно появиться сообщение об успешной настройке: время, дни, имя задания.

Проверка вручную: «Планировщик заданий» → «Библиотека планировщика заданий» → задача `OrderStatements_Rosreestr`.

---

### Изменить время или дни

1. Отредактируйте `time` и/или `days` в `schedule.ini`.
2. Снова запустите **`setup_schedule.bat`** (желательно от администратора).

Изменения в `schedule.ini` **сами не применяются** — всегда нужен повторный запуск `setup_schedule.bat`.

---

### Выключить автозапуск

**Способ 1 — быстрый**

Запустите **`disable_schedule.bat`**. Он:

- пропишет `enabled=false` в `schedule.ini`;
- удалит задание из планировщика.

**Способ 2 — вручную**

1. В `schedule.ini` поставьте `enabled=false`.
2. Запустите `setup_schedule.bat`.

Расписание отключено, но `time` и `days` в файле сохраняются — при следующем `enabled=true` не нужно вводить время заново.

---

### Снова включить после отключения

1. В `schedule.ini`: `enabled=true` (время и дни уже могут быть настроены).
2. Запустите `setup_schedule.bat`.

---

### Как работает запуск по расписанию

```
Планировщик Windows (в заданное время)
        ↓
  Пробуждение ПК из сна (если спит)
        ↓
  run_scheduled.bat
        ↓
  python -m script.main
        ↓
  logs\scheduled_run.log  +  обычные логи (errors.log, session_*.md, …)
```

`run_scheduled.bat` не показывает окно с `pause` — удобно для ночного запуска. Итог пишется в `logs\scheduled_run.log`.

## Структура папок и логов

```
Order_statements\
├── start.bat              ← ручной запуск
├── setup_schedule.bat     ← применить расписание

├── results.log            ← сводный лог результатов по файлам

│   ├── scheduled_run.log  ← лог автозапусков

│   ├── diagnostics\       ← HTML/JSON при ошибках
│   └── screenshots\
└── script\                ← исходный код Python
```

## Ограничения и требования

**Пробуждение из сна**

- Работает, если ПК в режиме **сна** или **гибернации**.
- **Не работает**, если компьютер **полностью выключен**.
- В BIOS/UEFI должны быть разрешены таймеры пробуждения (RTC Alarm / Wake on timer).
- `setup_schedule.bat` от администратора включает таймеры пробуждения в текущей схеме питания.

**Сеанс пользователя**

- Скрипт управляет Chrome на рабочем столе — нужен **вход в Windows** под вашей учётной записью (не экран блокировки на момент запуска).
- Профиль Chrome должен содержать актуальную сессию на сайте Росреестра.

**КриптоПро и диалоги Windows**

- Для подписания используется КриптоПро; нативные окна обрабатываются автоматически (`pywinauto`). При сбоях смотрите `logs\errors.log`.

## Частые вопросы

**Изменил `schedule.ini`, но время не поменялось**  
Нужно снова запустить `setup_schedule.bat` — файл только хранит настройки, в планировщик их переносит bat-файл.

**Как проверить, что задание создано**  
Откройте «Планировщик заданий» → `OrderStatements_Rosreestr`. В свойствах на вкладке «Условия» должно быть включено «Пробуждать компьютер для выполнения задачи».

**ПК не просыпается**  
Запустите `setup_schedule.bat` от администратора; проверьте настройки сна в Windows и BIOS; убедитесь, что ПК не выключен, а именно в сне/гибернации.

**Автозапуск сработал, но скрипт упал**  
Откройте `logs\scheduled_run.log`, затем `logs\errors.log` и последний `logs\session_*.md`.

**Чем отличается `start.bat` от `run_scheduled.bat`**  
`start.bat` — для ручной работы (есть `pause` в конце). `run_scheduled.bat` — для планировщика, пишет в `scheduled_run.log` и сразу завершается.

**Как временно не трогать расписание, но не запускать скрипт**  
Положите пустую папку `uploads_CSV` или уберите CSV — скрипт запустится, но нечего будет обрабатывать. Надёжнее — `disable_schedule.bat` или `enabled=false`.

## Краткая шпаргалка

| Задача | Действие |
|--------|----------|
| Запустить сейчас | `start.bat` |
| Включить автозапуск в 8:30 по будням | `schedule.ini`: `enabled=true`, `time=08:30`, `days=weekdays` → `setup_schedule.bat` (админ) |
| Поменять время на 9:00 | `time=09:00` в `schedule.ini` → `setup_schedule.bat` |
| Запускать только по понедельникам | `days=пн` → `setup_schedule.bat` |
| Выключить автозапуск | `disable_schedule.bat` |
| Включить снова | `enabled=true` в `schedule.ini` → `setup_schedule.bat` |
